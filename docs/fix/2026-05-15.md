## 문서 개선

### 개선 1: 서비스 흐름 리뷰 현재 버전 반영

**변경 전**

`report/service_flow_review.md`는 2026-04-20 기준의 query/synthesis 중심 리뷰였다. 현재 코드에 추가된 멀티 도메인 저장 구조, source registry, 추출 캐시, 구조화 ingest, 최신 테스트 상태가 반영되어 있지 않았다.

**변경 후**

현재 버전 기준의 전체 서비스 흐름 리뷰로 재작성했다.

반영 내용:

- 시스템 아키텍처를 Frontend, Web Backend, Ops/Core, LLM Adapter, Storage, Search 레이어로 재정리
- `wiki/{domain}`와 `data/{domain}` 분리 저장 구조 설명
- PDF 원본 저장 위치와 `sources.jsonl`, `.cache/` 구조 설명
- 문서 업로드 흐름 추가
- Ingest 흐름을 source registry, 구조화 추출, fallback, index/log 업데이트까지 반영
- PDF 읽기/캐시 흐름 추가
- “소스 파일 읽는 중...” 단계가 오래 걸리는 이유 정리
- Query, Synthesis, Markdown 렌더링, Lint 흐름을 현재 구현 기준으로 갱신
- 현재 완료/부분 완료/미착수 개선 상태 정리
- 다음 개선 제안으로 source page template rendering, page별 evidence 전달, search index cache, deterministic lint 제시

**수정 파일**

| 파일 | 수정 내용 |
|---|---|
| `report/service_flow_review.md` | 현재 버전 기준 서비스 흐름 리뷰로 전면 갱신 |
