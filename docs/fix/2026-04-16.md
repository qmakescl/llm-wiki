## 버그 수정

### 버그 1: 첫 실행 시 빌드 백엔드 오류

**현상**

`./launch.sh` 첫 실행 시 패키지 설치 단계에서 오류 발생:

```
pip._vendor.pyproject_hooks._impl.BackendUnavailable: Cannot import 'setuptools.backends.legacy'
```

**원인**

`pyproject.toml`에 지정된 빌드 백엔드(`setuptools.backends.legacy:build`)가 setuptools 68.3.2 이상에서만 사용 가능한 신규 API였음. 신규 가상환경에는 setuptools가 없거나 구버전이어서 import 시점에 실패.

**해결**

```toml
# 수정 전
build-backend = "setuptools.backends.legacy:build"

# 수정 후
build-backend = "setuptools.build_meta"
```

`setuptools.build_meta`는 setuptools 40+ 전 버전에서 동작하는 표준 백엔드. `wheel`을 빌드 의존성에 추가해 editable install 안정성 확보.

**수정 파일**: `pyproject.toml`

### 버그 2: 실패한 가상환경 재사용으로 재실행 불가

**현상**

설치 실패 후 `.venv` 디렉토리가 남은 상태에서 재실행하면 설치 단계를 건너뛰어 앱이 시작되지 않음.

**원인**

`launch.sh`가 `.venv` 디렉토리 존재 여부만 확인해 실패한 환경을 재사용.

**해결**

```bash
# 수정 전
if [ ! -d ".venv" ]; then ...

# 수정 후
if [ ! -d ".venv" ] || ! .venv/bin/python -c "import wiki_web, fastapi, uvicorn" 2>/dev/null; then
    rm -rf .venv
    python3 -m venv .venv
    .venv/bin/pip install --upgrade pip setuptools wheel -q
    .venv/bin/pip install -e . -q
fi
```

**수정 파일**: `launch.sh`

### 버그 3: 웹 의존성 optional extras로 분리되어 누락

**현상**

```
ModuleNotFoundError: No module named 'fastapi'
```

**원인**

`fastapi` 등 웹 패키지가 `[project.optional-dependencies]` `web` 섹션에만 있어, `uv add` 등 외부 도구 사용 시 extras가 자동 설치되지 않아 누락됨.

**해결**

웹 의존성(`fastapi`, `uvicorn[standard]`, `python-multipart`, `jinja2` 등)을 메인 `dependencies`로 통합, `web` extras 섹션 제거.

**수정 파일**: `pyproject.toml`, `launch.sh`

### 버그 4: Starlette 0.36+ TemplateResponse API 변경

**현상**

```
TypeError: unhashable type: 'dict'
  File ".../jinja2/utils.py", line 515, in __getitem__
```

**원인**

Starlette 0.36+에서 `TemplateResponse` 시그니처 변경. 구 방식으로 호출하면 dict가 템플릿 이름으로 전달되어 Jinja2 캐시 키 조회 실패.

```python
# 구 API
templates.TemplateResponse("template.html", {"request": request, ...})

# 신 API
templates.TemplateResponse(request, "template.html", {...})
```

**해결**

모든 라우터의 `TemplateResponse` 호출을 신 API로 변환.

**수정 파일**: `wiki_web/routers/` 하위 5개 파일 (wiki, documents, query, settings, lint)
