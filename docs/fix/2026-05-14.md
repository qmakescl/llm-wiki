### 개선 4: pytest 기본 수집 범위 안정화

**현상**

기본 `pytest` 실행 시 `release/2026-04-21/tests` 복제본까지 수집되어 `tests.conftest` import mismatch가 발생했다.

**해결**

`pyproject.toml`에 pytest 수집 경로를 명시했다.

```toml
[tool.pytest.ini_options]
testpaths = ["tests"]
```

|---|---|
| `pyproject.toml` | pytest 기본 수집 범위를 `tests`로 고정 |

### 개선 5: Source Registry 및 hash 기반 중복 추적

**배경**

기존 중복 검사는 `sources/{slug}.md` 존재 여부 중심이라, 같은 파일을 다른 이름으로 업로드하면 중복 ingest를 막기 어려웠다.

**변경 내용**

- `data/{domain}/sources.jsonl` registry 추가
- source별 metadata 기록
  - `source_id`
  - `relative_path`
  - `filename`
  - `sha256`
  - `size`
  - `uploaded_at`
  - `ingested_at`
  - `summary_page`
  - `model`
- 웹 업로드 시 source registry 등록
- ingest 완료 시 registry에 ingest 상태 기록
- 같은 내용의 이미 ingest된 source가 있으면 LLM 호출 전에 중단

**수정 파일**

| 파일 | 수정 내용 |
|---|---|
| `wiki_cli/source_registry.py` | source registry 신규 구현 |
| `wiki_cli/ops/ingest.py` | sha256 기반 중복 ingest 검사 및 ingest 완료 기록 연결 |
| `wiki_web/routers/documents.py` | 업로드 시 registry 등록, ingest에 `data_root` 전달 |
| `tests/test_source_registry.py` | registry 생성/갱신/중복 감지/ingest 차단 테스트 추가 |

### 개선 6: PDF/text 추출 캐시와 LLM file-result 캐시

**배경**

PDF 업로드 후 ingest 재시도 시 텍스트 추출과 첫 LLM 분석을 반복하는 비용이 컸다.

**변경 내용**

- `llm.call_with_file()`에 extracted text cache 추가
- 같은 파일 sha256, prompt/system hash, model/chunk 설정이면 LLM 결과 재사용
- 캐시 위치
  - `data/{domain}/.cache/extracted_text/`
  - `data/{domain}/.cache/llm_results/`
  - ad-hoc CLI 파일은 파일 옆 `.llm_wiki_cache/`
- 캐시 제어 환경변수 추가
  - `WIKI_EXTRACT_CACHE=0`
  - `WIKI_LLM_FILE_CACHE=0`

**수정 파일**

| 파일 | 수정 내용 |
|---|---|
| `wiki_cli/llm.py` | 추출 캐시와 `call_with_file()` 결과 캐시 추가 |
| `tests/test_llm_runtime.py` | 캐시 root, 추출 캐시, LLM 결과 캐시, 캐시 비활성화 테스트 추가 |

### 개선 7: 구조화 ingest 1차 도입

**배경**

기존 ingest는 자유서술 overview를 생성한 뒤 다시 LLM을 호출해 entity/concept 계획을 세웠다. 같은 내용을 반복 해석하는 비용이 있었다.

**변경 내용**

- `StructuredIngestResult` JSON 추출 도입
- 구조화 결과 필드
  - `summary`
  - `claims[]`
  - `entities[]`
  - `concepts[]`
  - `uncertainties[]`
  - `contradiction_candidates[]`
- 구조화 파싱 성공 시 `entities[]`, `concepts[]`에서 page plan 생성
- `_plan_related_pages()` LLM 호출 생략
- 구조화 파싱 실패 시 기존 overview 기반 흐름으로 fallback

**개선 확인**

테스트 fixture 기준:

| 항목 | 기존 흐름 | 구조화 성공 흐름 |
|---|---:|---:|
| source 분석 `call_with_file()` | 1 | 1 |
| entity/concept planning `llm.call()` | 1 | 0 |
| source page 생성 `llm.call()` | 1 | 1 |
| entity page 생성 `llm.call()` | 1 | 1 |
| concept page 생성 `llm.call()` | 1 | 1 |
| 총 LLM 호출 | 5 | 4 |

**수정 파일**

| 파일 | 수정 내용 |
|---|---|
| `wiki_cli/structured_ingest.py` | 구조화 ingest prompt, parser, overview renderer 추가 |
| `wiki_cli/ops/ingest.py` | 구조화 추출 연결 및 planning LLM 호출 생략 |
| `tests/test_structured_ingest.py` | parser, entries 변환, overview 렌더링, ingest 구조화 경로 테스트 추가 |
